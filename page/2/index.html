中文说明1

安装

安装 kubebuilder

os=$(go env GOOS)
arch=$(go env GOARCH)

# 下载 kubebuilder 并解压到 tmp 目录中
curl -L -o kubebuilder https://go.kubebuilder.io/dl/latest/$(go env GOOS)/$(go env GOARCH)
chmod +x kubebuilder && mv kubebuilder /usr/local/bin

创建一个项目

$ mkdir $GOPATH/src/kubehello
$ cd $GOPATH/src/kubehello
$ kubebuilder init --domain kubehello.local --repo kubehello.local/guestbook

配置Registry的密钥

$ kubectl create secret docker-registry regcred --namespace=kubehello-system --docker-server=**** --docker-username=**** --docker-password=**** --docker-email=stdhi@grepcode.cn

创建一个 API

$ kubebuilder create api --group webapp --version v1 --kind Guestbook

测试

$ make install
$ make run

配置Registry授权

编辑 ./config/manager/manager.yaml

...
    spec:
      securityContext:
        runAsNonRoot: true
      imagePullSecrets:
        - name: regcred
      containers:
      - command:
...

安装Resource

$ kubectl apply -f config/samples/

在Cluster运行

# 打包Example镜像
$ make docker-build docker-push IMG=hm.grepcode.cn:55582/guestbook:latest

# 部署Example镜像
$ make deploy IMG=hm.grepcode.cn:55582/guestbook:latest

删除CRDs

$ make uninstall

卸载Controller

$ make undeploy

1.1 参考文档

cert-manager官方文档

证书管理器
cert-manager 将证书和证书颁发者添加为 Kubernetes 集群中的资源类型，并简化了获取、更新和使用这些证书的过程。

它可以从各种受支持的来源颁发证书，包括 Let's Encrypt、HashiCorp Vault 和 Venafi 以及私有 PKI。

它将确保证书有效且最新，并尝试在到期前的配置时间续订证书。

它大致基于 kube-lego 的工作，并借鉴了其他类似项目（例如 kube-cert-manager）的一些智慧。
